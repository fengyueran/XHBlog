记得曾经看到过这样一道面试题：

console 的输出顺序是什么？

```
setTimeout(() => {
  console.log("1111");
}, 0);

console.log("2222");

new Promise((resolve, reject) => {
  console.log("3333");
  resolve()
}).then(() => {
  console.log("4444");
});
```

想要判断 console 的输出顺序，必须了解 JavaScript 的执行机制，不然只能靠感觉去猜，陷入深深的迷茫中 😇😇😇...

众所周知 JavaScript 是一门**单线程语言**，也就意味着同一时间只能完成一个任务。为了防止一些耗时的任务(如加载图片、ajax 请求)影响我们的体验，就需要一种机制去调度任务，这个机制就是 event loop(事件循环)。
